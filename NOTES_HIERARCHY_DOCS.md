# Иерархия заметок и связь с задачами

## Обзор

Система заметок теперь поддерживает:
- **Вложенность заметок** с контролем уровня рекурсии (максимум 10 уровней)
- **Связь заметок с задачами** для организации информации по проектам
- **Автоматическое создание заметки** из описания задачи при её создании

## Структура базы данных

### Таблица `notes`

Новые поля:
- `parent_note_id` (UUID, nullable) - ID родительской заметки для вложенности
- `nesting_level` (INTEGER, default: 0) - уровень вложенности (0-10)
- `task_id` (UUID, nullable) - ID связанной задачи

### Связи
- `Notes.parentNote` -> `Notes` (Many-to-One)
- `Notes.childNotes` -> `Notes[]` (One-to-Many)
- `Notes.task` -> `Task` (Many-to-One)
- `Task.notes` -> `Notes[]` (One-to-Many)

## API Endpoints

### Заметки

#### POST `/notes`
Создание новой заметки с поддержкой вложенности

```json
{
  "name": "Название заметки",
  "text_content": "Содержание заметки",
  "parent_note_id": "uuid-родительской-заметки", // опционально
  "task_id": "uuid-задачи", // опционально
  "nesting_level": 1 // опционально, вычисляется автоматически
}
```

#### GET `/notes/me`
Получение всех заметок пользователя с пагинацией

#### GET `/notes/:id`
Получение заметки по ID с информацией о родителе и детях

#### GET `/notes/:id/hierarchy`
Получение заметки со всей иерархией потомков

#### GET `/notes/task/:taskId`
Получение всех заметок, связанных с задачей

#### PATCH `/notes/:id`
Обновление заметки с валидацией вложенности

#### DELETE `/notes/:id`
Удаление заметки (дочерние заметки удаляются каскадно)

### Задачи

#### GET `/tasks/:task_id/notes`
Получение всех заметок, связанных с задачей

## Логика работы

### Контроль вложенности

1. **Максимальный уровень**: 10 уровней вложенности
2. **Автоматический расчет**: При указании `parent_note_id` уровень вычисляется автоматически
3. **Валидация циклических ссылок**: Предотвращение создания циклов в иерархии
4. **Каскадное удаление**: При удалении родительской заметки все дочерние удаляются

### Связь с задачами

1. **Автоматическое создание**: При создании задачи с описанием автоматически создается заметка "Task Description"
2. **Синхронизация**: При обновлении описания задачи соответствующая заметка обновляется
3. **Контроль доступа**: Пользователь может видеть заметки только тех задач, участником которых является

## Примеры использования

### Создание вложенной заметки

```bash
# Создание родительской заметки
POST /notes
{
  "name": "Планирование проекта",
  "text_content": "Основные этапы разработки"
}

# Создание дочерней заметки
POST /notes
{
  "name": "Этап 1: Анализ",
  "text_content": "Требования и техническое задание",
  "parent_note_id": "uuid-родительской-заметки"
}
```

### Связывание заметки с задачей

```bash
POST /notes
{
  "name": "Заметка по задаче",
  "text_content": "Дополнительная информация",
  "task_id": "uuid-задачи"
}
```

### Получение иерархии заметок

```bash
GET /notes/{id}/hierarchy
```

Возвращает заметку со всеми дочерними заметками в виде дерева.

## Миграция базы данных

Для применения изменений запустите миграцию:

```bash
npm run migration:run
```

Миграция добавит новые поля в таблицу `notes` и создаст необходимые индексы и внешние ключи.

## Ограничения и особенности

1. **Максимальная вложенность**: 10 уровней
2. **Каскадное удаление**: При удалении родительской заметки или задачи связанные заметки удаляются
3. **Валидация доступа**: Пользователь может работать только со своими заметками
4. **Автоматическая синхронизация**: Описание задачи автоматически синхронизируется с заметкой

## Рекомендации по использованию

1. Используйте иерархию для структурирования информации по проектам
2. Связывайте заметки с задачами для контекстной организации данных
3. Не создавайте слишком глубокую вложенность (рекомендуется до 5 уровней)
4. Используйте понятные названия для заметок в иерархии
