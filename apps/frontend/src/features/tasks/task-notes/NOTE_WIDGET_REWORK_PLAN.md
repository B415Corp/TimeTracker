# План рефакторинга заметок (аналог Notion): хранение структуры строк как JSON

## Архитектурная суть
- **Заметка** — это сущность, в которой поле `content` (или `text_content`) хранит JSON-структуру всех блоков (строк) заметки.
- **Блок заметки** (NoteBlock/NoteLine) — элемент массива внутри JSON: `{ id, parentId, order, type, content, ... }`.
- Любой блок может быть вложенным, иметь произвольный тип (заголовок, список, todo, цитата, код, файл, divider и т.д.), содержать rich content.
- Вся вложенность, порядок, типы, контент и доп. параметры блоков хранятся внутри одного поля заметки.
- На фронте блоки отображаются и редактируются с поддержкой вложенности, drag-and-drop между уровнями, типов, rich content, быстрых действий и т.д. (UX как в Notion).

## Пример структуры JSON в заметке (Notion-like)
```json
[
  { "id": "1", "parentId": null, "order": 0, "type": "heading", "content": "Заголовок" },
  { "id": "2", "parentId": null, "order": 1, "type": "text", "content": "Текст" },
  { "id": "3", "parentId": "2", "order": 0, "type": "list", "content": "Пункт" },
  { "id": "4", "parentId": null, "order": 2, "type": "todo", "content": "Сделать задачу", "checked": false },
  { "id": "5", "parentId": null, "order": 3, "type": "divider" },
  { "id": "6", "parentId": null, "order": 4, "type": "code", "content": "console.log('hi')", "lang": "js" }
]
```

---

## Примечания
- Вся логика вложенности, порядка и типов теперь только внутри JSON.
- Бэкенд не управляет отдельными блоками — только целиком JSON.
- Архитектура и UX максимально приближены к Notion (гибкость, future-proof).
- Миграция обязательна для всех старых заметок.
